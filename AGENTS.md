# AGENTS.md

## Project Overview

Go HTTP API exposing two endpoints:
- `GET /echo` and `POST /echo` — accepts a message and echoes it back as JSON
- `GET /stats` — returns current UTC server time, OS memory usage, and Go runtime heap stats

Full OpenAPI documentation is served via Swagger UI at `/swagger/index.html`.

## Prerequisites

### Installing the Latest Go Compiler

**macOS (Homebrew) — recommended:**
```sh
brew install go
# or upgrade an existing installation:
brew upgrade go
```

**Linux (official tarball):**
```sh
# Replace <VERSION> with the latest from https://go.dev/dl/ (e.g. 1.22.4)
wget https://go.dev/dl/go<VERSION>.linux-amd64.tar.gz
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf go<VERSION>.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.profile
source ~/.profile
```

**Verify:**
```sh
go version
# expected: go version go1.22.x ...
```

### Installing the swag CLI (one-time, required for doc regeneration)

```sh
go install github.com/swaggo/swag/cmd/swag@latest
# Adds `swag` to $(go env GOPATH)/bin
# Ensure that directory is in your PATH:
export PATH="$PATH:$(go env GOPATH)/bin"
```

## Development Workflow

### Run locally

```sh
go run ./main.go
# Server starts on :8080 by default
```

Set `PORT` or `GIN_MODE` as needed:

```sh
GIN_MODE=release PORT=9000 go run ./main.go
```

### Regenerate OpenAPI docs

**Required after any `// @...` annotation change:**

```sh
swag init -g main.go --output docs
# or use the helper script:
./scripts/swag-init.sh
```

### Build a binary

```sh
go build -o hello-world-tasks ./...
```

### Run all tests

```sh
go test ./...
```

### Run tests with verbose output

```sh
go test ./internal/handler/... -v
```

### Run tests with coverage report

```sh
go test ./internal/handler/... -coverprofile=coverage.out
go tool cover -html=coverage.out          # opens browser
go tool cover -func=coverage.out          # prints per-function summary
```

### Run a specific test by name

```sh
go test ./internal/handler/... -run TestEchoHandler_GET_ReturnsMessage
go test ./internal/handler/... -run TestAPI_Stats
```

### Test file locations

| File | Package | Purpose |
|---|---|---|
| `internal/handler/echo_test.go` | `package handler` | Unit tests for `EchoHandler` |
| `internal/handler/stats_test.go` | `package handler` | Unit tests for `StatsHandler` |
| `internal/handler/api_test.go` | `package handler_test` | Black-box API integration tests via `httptest` |

### CI spec-drift check

Detect if annotations were changed without regenerating `docs/`:

```sh
swag init -g main.go --output docs
git diff --exit-code docs/
```

## Coding Conventions

- JSON field names use `snake_case`
- All 4xx/5xx responses use the shared `ErrorResponse` struct from `internal/handler/handler.go`
- Use `ShouldBind` (not `ShouldBindJSON`) for handlers that accept both GET and POST so that query-string and JSON body bindings are resolved by content type automatically
- All exported request/response types must have a `// @Description` comment for Swagger
- Never edit `docs/` by hand — always regenerate with `swag init`
- New endpoints: add the handler file under `internal/handler/`, then register the route in `RegisterRoutes()` in `handler.go`
